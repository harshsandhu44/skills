# Issue Writer

Use this skill to write a clear issue from rough input.

## Goals

- Make the issue understandable and actionable
- Capture the problem, impact, and expected outcome
- Clarify scope without over-specifying implementation

## Instructions

1. Read the rough input.
2. Identify:
   - problem or request
   - affected users or systems
   - impact
   - desired outcome
   - scope boundaries
3. Choose the right issue type:
   - bug
   - feature
   - improvement
   - task
4. Write the issue clearly and directly.
5. Add acceptance guidance if useful.

## Output format

### Title

- Concise and specific

### Summary

- What the issue is and why it matters

### Problem / context

- Relevant background
- Who or what is affected

### Scope

- What should be addressed
- What is not in scope if relevant

### Expected outcome

- What success looks like

### Notes

- Dependencies, risks, assumptions, or relevant references

## Constraints

- Do not make the issue vague
- Do not overdesign the solution unless requested
- Do not omit why the work matters
