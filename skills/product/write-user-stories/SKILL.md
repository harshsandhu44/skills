# Write User Stories

Use this skill to convert rough product ideas into clear user stories.

## Goals

- Turn vague requests into implementable user stories
- Keep stories user-centered and outcome-focused
- Make scope clear enough for engineering and QA
- Avoid mixing multiple features into one story when they should be separated

## Instructions

1. Identify:
   - target user or persona
   - problem or need
   - desired outcome
2. Split the request into stories only where there is a real scope boundary.
3. Write each story in a simple, consistent format.
4. Add brief context where needed to clarify intent.
5. Keep stories small enough to be actionable.

## Preferred story format

As a <user>,
I want <capability>,
so that <outcome>.

## Output format

### Context

- Brief description of the problem or feature area

### User stories

For each story include:

- Story title
- User story statement
- Notes or scope clarification
- Dependencies if obvious

### Non-goals

- Things explicitly out of scope

## Constraints

- Do not write implementation details as if they are user needs
- Do not collapse unrelated functionality into one story
- Do not make the stories so vague that they are unusable
